Git

I've learned a little bit about Git, which is what I'll be using to store and share my code. Git is a version control system that you can read about more here. For us amateurs, a simple illustration will show you why such a system is so important. Let's say you are writing a book - an awesome book - and you are conflicted about how you want to end it. Does she die? Does he live? Happily ever after? Burn everything to the ground? What about the dog? After a few drinks, and a long stint of writer's block, you get home grab your laptop and the perfect ending just starts flowing out of you, almost faster than you can type it, you tweak the necessary details to tailor it perfectly, and finally your awesome book is complete. The next morning - no, the next afternoon, when you are finally feeling up to it, you decide to review your masterpiece, only to realize it isn't exactly a masterpiece. You've changed names, locations, and events just for the ending. You're worse off than when you started.

With Git, this nightmare can be avoided. Some documents are simple and don't depend on any others. Writing code, like writing a story, can become complex. A change to one document can impact others. In an effort to optimize and create more functionality, errors are often made. The ability to retrace our steps, and maintain working historical and alternate versions of a project is invaluable.

I won't go over the installation, you can find that here.

First, I had to initiate Git in my project directory, which holds nothing but a single HTML document. That's simple enough, just open up a terminal window, navigate to my project directory, and type in the command 'git init'. This gave me the following response 'Initialized empty Git repository inâ€¦'

Next step, run 'git add .', which will stage the new file, index.html, to be committed. Finally, I ran git commit -m 'Initial Commit' to create my first commit. The '-m' flag is for the message you want to describe your commit.

Github

In addition to helping an individual maintain code, Git is also very useful in allowing multiple people to collaborate on a single project, along with Github, sharing code with anyone who may be interested is very easy.

In addition to helping an individual maintain code, Git is also very useful in allowing multiple people to collaborate on a single project, along with Github, sharing code with anyone who may be interested is very easy.

I followed the directions here to push my local repository or repo, what the cool kids call it, to Github. It was pretty simple, while in my local repository I ran the following:

  1. git remote add origin https://github.com/AmateurCode/AmateurCode-Blog.git
  2. git push -u origin master

I was prompted for my Github login credentials, and boom I have a public repository you can check out here. On Github you can view the individual files in the repo, as well as clone the repository to check it out locally on your computer.
